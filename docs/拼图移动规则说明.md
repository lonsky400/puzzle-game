# 拼图游戏 — 已实现的移动规则说明

本文档完整梳理当前游戏中已实现的拼图移动规则（引擎 + 交互层）。

---

## 一、交互层：如何触发一次「移动」

### 1.1 两种操作方式

| 方式 | 触发条件 | 含义 |
|------|----------|------|
| **点击模式** | 先点击格子 A（选中），再点击格子 B | 以 A 所在格为起点、B 所在格为落点，执行一次 `swap(A, B)` |
| **拖拽模式** | 在格子 A 按下，拖动超过约 10px 后释放在格子 B | 以按下格 A 为起点、释放格 B 为落点，执行一次 `swap(A, B)` |

- 若未拖动（移动距离 ≤ 10px）则按**点击模式**处理（选中或与已选格交换）。
- 落点 `toPos` 由**释放时鼠标/手指所在的格子**决定；起点 `fromPos` 为**按下时所在格子**。
- 一次操作只调用一次 `engine.swap(fromPos, toPos)`，成功则 `onMove()` 并可能 `onWin()`。

### 1.2 参与移动的「组」的确定

- **起点格**：`fromPos`，其上拼图块 `fromPieceId = positionMap[fromPos]`。
- **主动组**：与 `fromPieceId` 在同一并查集内的所有块，即 `engine.uf.getGroupMembers(fromPieceId)`。
- 整次移动中，**主动组**以**整体**参与：形状不变，用同一偏移量 `(rowOff, colOff)` 计算目标格。

### 1.3 前置条件（不满足则不执行交换）

- 游戏未结束（`!engine.isComplete`）。
- 起点、落点均在网格内（`fromPos !== -1`, `toPos !== -1`）。
- 起点格有块（`fromPieceId !== -1`）。
- 起点与落点不是同一格（`toPos !== startPos` 时才会调用 `doSwap`）。
- 落点格上的块与起点格上的块**不属于同一组**（`!engine.uf.connected(fromPieceId, toPieceId)`）；同组内不交换。

---

## 二、引擎入口：swap 与 fallback

### 2.1 唯一入口

- 所有移动都通过 `engine.swap(fromPos, toPos)` 进入。
- `swap` 内部直接调用 `swapWithFallback(fromPos, toPos)`，无其他分支。

### 2.2 前置校验与分支（在 swapWithFallback 内）

1. **起点**  
   - `fromPos` 上必须有块（`fromPieceId !== -1`），否则直接返回 false。
2. **落点为空：滑入空位（经典滑动拼图）**  
   - 若 `toPos` 为空（`positionMap[toPos] === -1`），走 **滑入空位** 逻辑（见下节），不要求落点有块、也不做组挤占。
3. **落点有块：组挤占**  
   - 若 `toPos` 有块且与起点同组，返回 false；否则继续组目标与偏移计算。
4. **主动组与偏移**  
   - 主动组 = `getGroupMembers(fromPieceId)`，原位置集合 = `fromPositions`。  
   - 偏移量：`rowOff = toRow - fromRow`，`colOff = toCol - fromCol`（由落点格与起点格计算）。
3. **越界钳位**  
   - 对 `fromPositions` 做 `clampGroupOffset(fromPositions, rowOff, colOff)`，得到钳位后的 `rowOff`、`colOff`，使整组目标不越界（吸附到最近合法偏移）。
4. **目标格集合**  
   - `newFromPositions = getGroupTargetPositions(fromPositions, rowOff, colOff)`。  
   - 若为 `null`（越界）或存在重复格（组内目标自交），则**不执行组移动**，改为执行 **单块交换** `swapSingle(fromPos, toPos)`。

## 三、滑入空位（经典滑动拼图规则）

当**落点格为空**时（例如组挤占后产生空位，或关卡设计为有一空槽），按「滑入空位」规则处理，对应关键帧场景：网格中有一个空槽，只有与空位**四方向相邻**的那一块可以移入空位。

### 3.0 触发条件

- `toPos` 为空（`positionMap[toPos] === -1`）。
- 起点有块（`fromPieceId !== -1`）。

### 3.0.1 合法条件（不满足则 return false）

- **仅单块可滑入**：起点所在组必须只有 1 块（`getGroupMembers(fromPieceId).length === 1`）。多块组不能整体滑入一个空位。
- **必须相邻**：起点格与空位格在网格上**四方向相邻**（上/下/左/右），即 `manhattan(fromPos, emptyPos) === 1`。

### 3.0.2 执行

- 将该块的 `currentPos` 设为空位 `toPos`；原起点格 `fromPos` 变为空（`rebuildPositionMap` 后 `positionMap[fromPos] === -1`）。
- 然后 `rebuildUnionFind`、`moveCount++`、`checkWin()`。

---

## 四、组分割挤占（Group-Segmented Evacuation）

当目标格有块且与起点不同组时，按「组分割挤占」规则执行。

### 4.1 步骤 1：碰撞识别（重叠区与被挤块）

- **目标格集合**：`newFromPositions`（主动组将占据的格子）。
- **被挤块（stripped / orphanedBlocks）**：满足以下条件的块  
  - 其**当前所在格**在 `newFromPositions` 中；且  
  - 不属于主动组（与 `fromPieceId` 不同组）。  
- 实现：遍历 `newFromPositions`，对每个格取 `pieceId = positionMap[pos]`，若存在且 `!uf.connected(pieceId, fromPieceId)`，则该 `pieceId` 加入 `stripped` 数组。
- **保留块**：被动组中**当前格不在** `newFromPositions` 里的块；这些块**不移动**，保留在原格。

### 4.2 步骤 2：组分割（逻辑上的处理）

- **被挤块**：从原组中「剥离」，视为独立块，只参与空位分配与位移；不显式改 Group ID，通过后续 `rebuildUnionFind` 自然形成新的根或独立根。
- **保留块**：位置不变；若与同组其他保留块在网格上仍四连通，则仍属同一组；若被挖成多段，则 `rebuildUnionFind` 后形成多个根（多个新组）。完全覆盖时，该被动组所有块都被挤走，原组自然消失。

### 4.3 步骤 3：空位池与无空位约束

- **空位池（evacuatedSlots）**：仅来自**主动组原位置**，即 `fromPositions`（排序后记为 `freeSlots`），大小 = 主动组块数 `|A|`。
- **无空位约束**（移动后不得出现空位）：  
  - 若 `stripped.length > freeSlots.length`：空位不足，**拒绝**（`return false`）。  
  - 若 `stripped.length !== freeSlots.length`：会产空位或放不下，**拒绝**（`return false`）。  
- 仅当 **被挤块数 === 主动组块数** 时执行组挤占，被挤块 1:1 填满主动组原位置，**移动后无空位**。

### 4.4 步骤 4：空位分配（1:1，最近优先）

- 在满足 `stripped.length === freeSlots.length` 的前提下：  
  - 为每个被挤块分配**恰好一个**空位（来自 `freeSlots`），且每个空位只分配给一块。  
  - 分配策略：按**曼哈顿距离最近优先**贪心——对当前待分配的块，选一个尚未被占用的空位，使其到该块当前格的曼哈顿距离最小；重复直到所有被挤块都分配到一个空位。
- 结果：被挤块与空位一一对应，所有主动组原位置被填满。

### 4.5 步骤 5：位置更新（执行顺序）

1. **被挤块**：每个被挤块的 `currentPos` 更新为其分配到的空位（`freeSlots` 中的某一格）。  
2. **主动组**：主动组内每块的 `currentPos` 更新为 `newFromPositions` 中对应位置（整体刚性移动，形状不变）。  
3. **保留块**：不修改 `currentPos`，仍在原格。

### 4.6 步骤 6：状态重建与合并检测

- `rebuildPositionMap()`：按所有块的 `currentPos` 重算 `positionMap[pos] = pieceId`（未占用的格保持 -1；在当前无空位约束下不应出现）。
- `rebuildUnionFind()`：重新建并查集后执行 `checkAllMerges()`（见下）。
- `moveCount++`，`checkWin()` 更新是否完成。

---

## 五、自动合并规则（checkAllMerges）

- 在每次 `rebuildPositionMap` 之后、`rebuildUnionFind` 内会调用 `checkAllMerges()`。
- **合并条件**：两个相邻格（上下或左右）上的两块，若在**原始图片**中也是相邻（行/列差为 1 的同一行或同一列），则合并为同一组（`uf.union`）。
- 只检查**右邻**和**下邻**，避免重复；合并会迭代直到无法再合并。
- 效果：被挤块放到新位置后，若与邻格在「原图」中相邻，会与新邻居自动成组；保留块若仍相邻则保持同组；被挖成多段的被动组会自然变成多个新组（不同根）。

---

## 六、单块交换（swapSingle）— 何时发生

仅在**组目标不合法**时执行，用于兜底一次「交换」：

- **触发条件**（在 `swapWithFallback` 内）：  
  - `newFromPositions === null`（偏移后组越界），或  
  - `newFromPositions` 中存在重复格（组内目标自交）。  
- **行为**：只交换**起点格**与**落点格**上的两块（`fromPos` ↔ `toPos` 的 `currentPos` 互换），不涉及其他格、不按组移动、不挤占。
- **前置**：两格有块且不同组（与主流程相同）。

---

## 七、边界与拒绝条件汇总

| 情况 | 行为 |
|------|------|
| 起点无块 | 不执行，`swap` 返回 false |
| 落点为空且（非单块或不相邻） | 不执行，`slideIntoEmpty` 返回 false |
| 落点为空且（单块且四方向相邻） | 执行滑入空位，该块移入空位、原格变空 |
| 落点有块且与起点同组 | 不执行，返回 false |
| 目标格越界（getGroupTargetPositions 返回 null） | 操作无效，返回 false |
| 组目标自交（newFromPositions 有重复） | 操作无效，返回 false |
| 被挤块数 > 主动组块数（空位不足） | 拒绝，返回 false |
| 被挤块数 ≠ 主动组块数（会产空位或放不下） | 拒绝，返回 false（保证移动后无空位） |
| 被挤块数 = 主动组块数 | 按组分割挤占执行，1:1 填满，移动后无空位 |

---

## 八、状态一致性（当前实现保证的约束）

1. **网格**：格子数 = 块数；在组挤占成功执行后，每个格至多一块（当前实现要求 1:1，故无空位）。
2. **并查集**：每个块属于恰好一个根；合并仅由 `checkAllMerges` 根据「当前格子相邻 + 原图相邻」更新。
3. **主动组移动**：整组用同一偏移，形状不变（刚性移动）。
4. **被动组**：仅重叠区内的块被移动；非重叠区块保留在原位，组关系由并查集重建结果体现。

## 九、与文档的对应关系

- **Untitled-1.ini / Untitled-2.ini**：采用「组分割挤占」——重叠区剥离为独立块、保留块不动、空位仅来自主动组原位置。
- **无空位**：通过强制 `stripped.length === freeSlots.length` 保证移动后无空位。
- **空位分配**：优先使用主动组原位置（本游戏唯一空位来源），按曼哈顿距离最近优先分配。
- **连环挤占 / 级联**：当前未实现；空位不足时直接拒绝，不回滚到单块交换（组目标合法时）。
