# 拼图游戏 — 已实现规则总览

本文档完整整理当前项目中**已实现**的拼图操作、移动规则、合并与拆分等行为，便于查阅与交接。

---

## 一、交互方式

### 1.1 两种操作

| 方式 | 触发 | 效果 |
|------|------|------|
| **点击** | 点击格 A（选中）→ 再点击格 B | 执行一次 `swap(A, B)`，起点 A、落点 B |
| **拖拽** | 在组内任意格 A 按下 → 拖动 >10px → 松手 | 若拖动偏移（格）在任一方向 **> 半格** 则执行一次 `swap(A, toPos)`，否则不移动 |

- 未拖动（位移 ≤10px）时按点击处理：若已有选中格且点击另一格则交换，否则选中/取消选中。
- **起点**：按下时鼠标/手指所在格 `fromPos`（可为主动组内任意格）。
- **拖拽时的目标**：与松手所在格无关。用主动组外接矩形**锚点（左上角）** + **拖动像素偏移**换算成格数偏移；仅当偏移在**任一方向 > 半格**时触发移动，目标格 = 锚点 + `round(偏移格数)`；若未超过半格则不移动。
- 每次操作至多调用一次 `engine.swap(fromPos, toPos)`；成功则 `onMove()`，若完成则 `onWin()`。

### 1.2 组的认定

- 起点格上的块：`fromPieceId = positionMap[fromPos]`。
- **主动组**：与 `fromPieceId` 同属一个并查集根的所有块，即 `engine.uf.getGroupMembers(fromPieceId)`。
- 整次移动中，主动组以**整体**参与：形状不变，用同一偏移量移动。

### 1.3 不执行的情况（交互层）

- 游戏已结束（`engine.isComplete`）。
- 起点格为空或越界（不应出现）。
- **拖拽**：偏移在行、列方向均 ≤ 半格时不移动；或目标格越界、或目标格与起点同一格时不调用 `doSwap`。
- **点击**：落点越界或与起点同一格时不交换。

## 二、引擎入口与分支

### 2.1 唯一入口

- 所有移动都通过 `engine.swap(fromPos, toPos)` 进入。
- `swap(fromPos, toPos)` 内部直接调用 `swapWithFallback(fromPos, toPos)`。

### 2.2 swapWithFallback 内部分支

1. **起点无块**  
   `positionMap[fromPos] === -1` → 返回 `false`。

2. **落点为空**  
   `positionMap[toPos] === -1` → 走 **滑入空位** 逻辑（见第三节），不参与组挤占。

3. **落点有块且与起点同组**  
   `uf.connected(fromPieceId, toPieceId)` → 返回 `false`（同组不交换）。

4. **落点有块且不同组**  
   走 **组整体移动 + 组分割挤占**（见第四节）。若越界、自交、空位约束不满足等，直接返回 `false`，**不**退化为单块交换。

---

## 三、滑入空位（slideIntoEmpty）

当**落点格为空**时使用，对应「网格中有一个空槽，相邻单块可滑入」的经典滑动拼图规则。

### 3.1 条件

- 落点为空：`toPos` 上 `positionMap[toPos] === -1`。
- **仅单块**：`getGroupMembers(fromPieceId).length === 1`，多块组不能整体滑入一个空位。
- **四邻**：起点与空位曼哈顿距离为 1（上下左右之一）。

### 3.2 执行

- 将该块的 `currentPos` 设为 `toPos`。
- `rebuildPositionMap()`、`rebuildUnionFind()`、`moveCount++`、`checkWin()`。

不满足上述条件时 `slideIntoEmpty` 返回 `false`。

---

## 四、组整体移动与组分割挤占（Group-Segmented Evacuation）

当落点有块且与起点不同组时，按「主动组整体移动 + 被动组重叠区被挤走、非重叠区保留」的规则执行。

### 4.1 主动组与偏移

- **主动组**：`fromGroup = getGroupMembers(fromPieceId)`，其当前格集合 `fromPositions`。
- **锚点**：组外接矩形的**左上角**  
  `anchorRow = min(行)`，`anchorCol = min(列)`，与用户点击组内哪一块无关。
- **原始偏移**：  
  由**锚点的位移量**决定，与松手所在格无关。UI 将拖动像素偏移换算成格数偏移（连续值），**仅当在任一方向 > 半格**时触发移动，取整为 `rowOff = round(offsetRow)`、`colOff = round(offsetCol)`，目标格 `toPos = (anchorRow + rowOff) * gridSize + (anchorCol + colOff)` 传给引擎；引擎内 `rowOff = toRow - anchorRow`、`colOff = toCol - anchorCol` 即该锚点偏移。
- **越界处理**：  
  不做偏移钳位。若 `getGroupTargetPositions(fromPositions, rowOff, colOff)` 为 `null`（整组目标越界），则**直接返回 false**，操作无效，保持原始位置。
- **目标格**：  
  `newFromPositions = getGroupTargetPositions(fromPositions, rowOff, colOff)`。  
  若为 `null`（越界）或存在重复格（组内目标自交），则**直接返回 false**，不执行单块交换。

### 4.2 碰撞：被挤块与保留块

- **目标格集合**：`newFromPositions`（主动组将要占用的格）。
- **被挤块（stripped）**：当前所在格在 `newFromPositions` 中，且与主动组不同组的块；这些块会被「剥离」为独立块，参与空位分配。
- **保留块**：被动组中当前格**不在** `newFromPositions` 里的块；**不移动**，保留在原格；连通性由之后的 `rebuildUnionFind` 体现（可能拆成多段、多组）。

### 4.3 空位池与挤压约束

- **空位池（freeSlots）**：主动组**离开且未再占**的格，即  
  `fromPositions` 中不在 `newFromPositions` 里的格：  
  `freeSlots = fromPositions.filter(p => !newFromSet.has(p))`。  
  当主动组整体移出原区域时，空位池即为主动组原先占用的全部格。
- **挤压约束**：  
  - `stripped.length > freeSlots.length` → 空位不足，无法安置全部被挤块，**拒绝**（返回 false）。  
  - `stripped.length <= freeSlots.length` → **允许执行**：被挤块填入空位池中的格（见下），主动组整体移入目标格；若被挤块数小于空位数，则未用到的空位保持为空，移动后可能产生空位。

### 4.4 空位分配（被挤块 → 空位池，按位置序一一对应）

- 被挤块与空位池均按网格位置 **(row, col)** 排序（行优先），然后 **stripped[i] → freeSlots[i]** 一一对应。
- 效果：目标区内「同一行/列」的被挤块会填到主动组原位置中「同一相对行/列」的空位，例如：原 20（目标区上方）→ 原 1 的位置，原 13（目标区下方）→ 原 6 的位置，组整体右移一列时保持「谁顶谁填顶、谁底谁填底」。
- 仅使用 `stripped.length` 个空位；`freeSlots` 中未被用到的格在移动后保持为空。

### 4.5 执行顺序

1. **被挤块**：每个被挤块的 `currentPos` 设为其分配到的空位（来自 `freeSlots` 中的格）；空位池中未被分配的格在重建后为空格。
2. **主动组**：主动组每块的 `currentPos` 设为 `newFromPositions` 中对应位置（整体刚性平移，形状与连续关系保持不变）。
3. **保留块**：不修改位置。

随后：`rebuildPositionMap()`、`rebuildUnionFind()`、`moveCount++`、`checkWin()`。

## 五、自动合并（checkAllMerges）

- 在每次 `rebuildUnionFind()` 内会调用 `checkAllMerges()`。
- **合并条件**：两个在网格上**相邻**的格（仅考虑右邻、下邻），若其上两块在**原图**中也是相邻（同行且列差 1，或同列且行差 1），则 `uf.union` 合并为同一组。
- 迭代直到无法再合并。
- **效果**：被挤块放到新位置后，若与原图邻居相邻会自动成组；保留块若仍相邻则保持同组；被「挖开」的被动组会自然变成多段、多个新组（并查集重建后体现）。

---

## 六、单块交换（swapSingle）

- **定义**：仅交换 `fromPos` 与 `toPos` 上两块的 `currentPos`，不按组移动、不挤占。
- **当前使用**：引擎内 **未** 在 `swapWithFallback` 中调用；越界或组目标自交时直接返回 false，**不退化为单块交换**。  
  `swapSingle` 仍存在于代码中，可供其他入口或测试使用。

---

## 七、边界与拒绝条件汇总

| 情况 | 行为 |
|------|------|
| 起点无块 | 返回 false |
| 落点为空且（非单块或不相邻） | 滑入空位不执行，返回 false |
| 落点为空且单块且四邻 | 执行滑入空位 |
| 落点有块且与起点同组 | 返回 false |
| 组目标越界（getGroupTargetPositions 为 null） | 返回 false |
| 组目标自交（newFromPositions 有重复） | 返回 false |
| 被挤块数 > 空位数 | 拒绝，返回 false（空位不足） |
| 被挤块数 ≤ 空位数 | 执行组分割挤占；被挤块与空位池按位置序一一对应填入（如原20→原1、原13→原6），未用空位保持为空，主动组整体移入目标并保持连续 |

## 八、其他已实现规则与状态

### 8.1 初始化与打乱

- **init()**：生成 `gridSize×gridSize` 块，初始 `currentPos = id`，然后 `shuffle()`，再 `rebuildPositionMap()`、`checkAllMerges()`。
- **shuffle()**：Fisher-Yates 随机排列位置；若正确块过多（> 约 20%）则重试，最多 100 次。

### 8.2 网格与状态约束

- 格子数 = 块数；组挤占执行后每格至多一块；当被挤块数 < 空位数时，空位池中未用到的格会变为空（`positionMap[pos] === -1`），绘制时已支持空格显示。
- 并查集：每块属于恰好一个根；合并仅由 `checkAllMerges` 根据「当前格相邻 + 原图相邻」更新。

### 8.3 边框与合并显示

- **shouldShowBorder(pos1, pos2)**：若两格有块且**不同组**则显示分隔线；同组则不画中间线（视觉上合并）。
- 绘制时：相邻且同组的块会消除间隙，形成「一块大块」的视觉效果。

### 8.4 胜利与进度

- **checkWin()**：所有块满足 `currentPos === id` 则 `isComplete = true`。
- **getProgress()**：正确块数 / 总块数，返回 0～100 的整数百分比。

### 8.5 移动次数与组数

- `moveCount`：每次成功 `swap`（含滑入空位、组挤占）加 1。
- `getGroupCount()`：当前并查集中不同根的数量，即当前「组」的数量。

## 九、与设计文档的对应

- **组分割挤占**：重叠区剥离为被挤块、保留块不动、空位池仅来自主动组离开且未再占的格（`fromPositions \ newFromPositions`）。
- **挤压规则**：仅当 `stripped.length > freeSlots.length` 时拒绝；`stripped.length <= freeSlots.length` 时执行，被挤块与空位池按位置序一一对应（谁在目标区上/左就填到空位池上/左），未用空位可保留，主动组整体移入目标并保持连续。
- **锚点偏移**：偏移由锚点位移量决定（UI 按拖动像素换算格数），与落点格无关；越界不钳位，整组目标越界则拒绝并保持原位置。
- **锚点**：以组外接左上角为锚点计算偏移，与点击组内哪一块无关。
- 越界或目标自交时**不**退化为单块交换，直接返回 false。

以上为当前拼图操作、移动规则、合并与拆分的已实现规则总览。
