# 你今天拼图了吗 — 游戏说明

## 一、游戏概述

**「你今天拼图了吗」** 是一款**可合并拼图**网页游戏：将打散的图块还原成完整图片。与经典滑动拼图不同，本作支持**多块成组移动**与**自动合并**——当相邻图块在原图中也相邻时，会自动合成一组，可整组拖拽，形成「越拼越大块」的体验。

- **技术栈**：React 19 + TypeScript + Vite，Canvas 绘制棋盘与图块，Framer Motion 做界面动效，国潮水墨风 UI。
- **运行方式**：前端 SPA（可配 Express 静态托管），支持本地开发与生产构建。

---

## 二、核心规则

### 2.1 目标

- 将一张图片切成的 **N×N** 块打散后，通过**移动**与**合并**，使每一块都回到其**正确位置**（即当前格 = 该块的原始 id 格）。
- 当所有块都归位时，游戏胜利。

### 2.2 图块与「组」

- 每个格子要么为空，要么有一块拼图；格子数等于总块数（存在空位时，部分格会空出）。
- **组**：通过并查集维护。若两块在**当前网格上相邻**（上下或左右），且在**原图中也相邻**，则自动合并为同一组。
- 同组图块在视觉上**无内部分割线**，可视为一块「大块」；移动时**整组一起动**，形状不变。

### 2.3 移动的两种情形

| 情形 | 条件 | 行为 |
|------|------|------|
| **滑入空位** | 落点格为空；起点为**单块**；起点与落点**四邻** | 该块滑到空位，一步完成。 |
| **组整体移动 + 挤占** | 落点格有块，且与起点**不同组** | 主动组整体平移；落点区域内的其他组块被「挤走」到主动组离开后留下的空位，按位置序一一对应；未重叠的被动组块保留不动。若被挤块数 > 空位数，则拒绝本次移动。 |

- **锚点**：组的外接矩形**左上角**。拖拽时目标位置由「锚点 + 拖动偏移」决定，与松手时鼠标所在格无关。
- **越界**：整组目标若超出棋盘，直接拒绝，不做自动钳位。
- 每次操作最多触发一次 `swap`；成功则步数 +1，并重新检测合并与胜利。

## 三、交互方式

### 3.1 点击模式

- 点击格 **A**：选中该格所在的一组（高亮朱砂红边框）。
- 再点击格 **B**：执行一次交换 `swap(A, B)`（A 为起点，B 为落点）；若 B 与 A 同组或无效则不执行。
- 再次点击已选中格：取消选中。

### 3.2 拖拽模式

- 在某一格**按下**（鼠标或触摸）→ 视为拾取该格所在组。
- **拖动超过约 10px** 后视为拖拽：目标由**锚点 + 像素偏移**换算成格偏移，仅当在**任一方向超过半格**时才执行移动。
- 松手时根据上述目标格执行一次 `swap(起点格, 目标格)`；若目标越界或未超过半格则不移动。
- 拖拽时：棋盘上该组显示为半透明 ghost，**跟随的图块**在**全屏 overlay** 上绘制，可拖出棋盘仍全屏跟随。

### 3.3 音效（Web Audio API 短音）

- **拾取**：按下有效图块时，极短、极轻的提示音。
- **放置**：每次成功放置（swap 成功）时，短促柔和音。
- **合并**：本次移动导致**新合并**时，播放一段清脆短和弦，与边界闪烁、图块放大抖动同步。

## 四、视觉与动效

### 4.1 风格

- **国潮水墨风**：宣纸暖白底色、墨色主文字、朱砂红强调（选中框、进度、按钮等）、可选水墨背景图。
- 棋盘为圆角矩形；格间有缝隙，**同组相邻块**在绘制时消除缝隙，视觉上连成一块。

### 4.2 合并反馈

- **边界亮光**：新合并的图块组沿其**真实外轮廓**（扩展边界含格缝，使相邻格顶点重合）描一圈亮光，多层描边 + 内层高光，约 480ms 内随 `1 - t²` 淡出。
- **放大抖动**：同一合并组在约前 20% 时间内从 1 放大到 1.06，20%～50% 时间内收回 1，与亮光同步，形成轻微「弹出」感。

### 4.3 其他

- 选中组：朱砂红边框 + 淡红填充。
- 拖拽时目标格（若与当前组不同组）：淡红高亮 + 虚线框。
- 胜利后：金色边框 + 胜利弹窗（步数、用时、烟花粒子、再来一局 / 返回关卡）。

---

## 五、关卡与难度

- **关卡**：多张预设图片（如齐天大圣、山水画卷、祥龙腾云、古街集市），每关对应一张图与一个默认网格尺寸（如 3×3、4×4）。
- **难度**：同一关卡内可通过**难度选择器**切换网格尺寸（如 3 / 4 / 5），改变块数；切换后重新生成打乱局面，步数与时长重置。
- **参考图**：游戏时可在侧边查看小图参考，辅助拼图。

---

## 六、胜利与进度

- **胜利条件**：所有块的 `currentPos === id`（即每块都回到自己的正确格）。
- **进度**：正确块数 / 总块数，以百分比显示，并有进度条。
- **统计**：步数（`moveCount`）、用时（秒）、当前组数（`getGroupCount()`）等，用于界面展示与胜利弹窗。

## 七、项目结构（简要）

- **`client/`**：前端 React 应用  
  - `src/pages/Home.tsx`：主页面（关卡选择、游戏中、胜利弹窗、计时与步数）。  
  - `src/components/PuzzleBoard.tsx`：棋盘 Canvas、拖拽 overlay、合并亮光与放大抖动、交互与音效触发。  
  - `src/lib/puzzleEngine.ts`：核心引擎（打乱、swap、滑入空位、组挤占、并查集、合并检测、胜利检测）。  
  - `src/lib/puzzleSound.ts`：拾取 / 放置 / 合并音效。  
- **`docs/拼图操作与移动规则总览.md`**：移动与合并规则的详细技术说明。  
- **`server/`**：可选 Express 静态托管与 API。
